# 통합 테스트 시나리오

이 문서는 Mattermost Reminder 서비스의 통합 테스트 시나리오를 제공합니다.

## 사전 준비

1. Firebase 프로젝트 설정 완료
2. Firestore 데이터베이스 활성화
3. Firebase Auth 활성화
4. Backend API 배포 완료
5. Frontend 개발 서버 실행 가능

## 테스트 환경 설정

### Backend
```bash
cd functions
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### Frontend
```bash
cd frontend
npm install
npm run dev
```

### 환경 변수
`frontend/.env.local`:
```env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_API_BASE_URL=http://localhost:8000
```

## 테스트 시나리오

### 시나리오 1: 사용자 인증 플로우

#### 1.1 회원가입
- [ ] `/login` 페이지 접근
- [ ] "회원가입" 클릭
- [ ] 이메일과 비밀번호 입력
- [ ] 회원가입 버튼 클릭
- [ ] 성공 시 대시보드로 리다이렉트
- [ ] Firebase Console에서 사용자 생성 확인

#### 1.2 로그인
- [ ] 로그아웃 상태에서 `/login` 접근
- [ ] 이메일과 비밀번호 입력
- [ ] 로그인 버튼 클릭
- [ ] 성공 시 대시보드로 리다이렉트
- [ ] 네비게이션에 이메일 표시 확인

#### 1.3 로그아웃
- [ ] 로그인 상태에서 "로그아웃" 버튼 클릭
- [ ] `/login` 페이지로 리다이렉트 확인
- [ ] 인증 상태 해제 확인

#### 1.4 인증 보호
- [ ] 로그아웃 상태에서 `/` 접근 시도
- [ ] 자동으로 `/login`으로 리다이렉트 확인
- [ ] 로그인 후 원래 페이지로 돌아가는지 확인

---

### 시나리오 2: Webhook 관리 플로우

#### 2.1 Webhook 추가
- [ ] 대시보드에서 "웹훅 관리" 클릭
- [ ] "웹훅 추가" 버튼 클릭
- [ ] 별칭 입력 (예: "개발팀 채널")
- [ ] Mattermost 웹훅 URL 입력
- [ ] "추가" 버튼 클릭
- [ ] 목록에 새 웹훅 표시 확인
- [ ] Firestore에서 데이터 저장 확인

#### 2.2 Webhook 목록 조회
- [ ] 웹훅 관리 페이지 접근
- [ ] 등록된 웹훅 목록 표시 확인
- [ ] 별칭, URL, 생성일 표시 확인

#### 2.3 Webhook 수정
- [ ] 웹훅 목록에서 "수정" 버튼 클릭
- [ ] 모달에서 별칭 또는 URL 수정
- [ ] "수정" 버튼 클릭
- [ ] 목록에 변경사항 반영 확인
- [ ] Firestore에서 업데이트 확인

#### 2.4 Webhook 삭제
- [ ] 웹훅 목록에서 "삭제" 버튼 클릭
- [ ] 확인 다이얼로그에서 확인
- [ ] 목록에서 제거 확인
- [ ] Firestore에서 삭제 확인

---

### 시나리오 3: Message 관리 플로우

#### 3.1 Message 추가
- [ ] "메시지 관리" 페이지 접근
- [ ] "메시지 추가" 버튼 클릭
- [ ] 메시지 내용 입력
- [ ] 요일 선택 (최소 1개)
- [ ] 시간 선택 (예: 09:00)
- [ ] 웹훅 선택
- [ ] 활성화 체크박스 선택
- [ ] "추가" 버튼 클릭
- [ ] 목록에 새 메시지 표시 확인
- [ ] Firestore에서 데이터 저장 확인

#### 3.2 Message 목록 조회
- [ ] 메시지 관리 페이지 접근
- [ ] 등록된 메시지 목록 표시 확인
- [ ] 메시지 내용, 요일, 시간, 상태 표시 확인

#### 3.3 Message 활성화/비활성화
- [ ] 메시지 목록에서 "활성화" 또는 "비활성화" 버튼 클릭
- [ ] 상태 변경 확인
- [ ] Firestore에서 `isActive` 필드 업데이트 확인

#### 3.4 Message 즉시 전송
- [ ] 메시지 목록에서 "즉시 전송" 버튼 클릭
- [ ] 확인 다이얼로그에서 확인
- [ ] 성공 메시지 표시 확인
- [ ] Mattermost 채널에서 메시지 수신 확인
- [ ] 메시지 내용이 정확한지 확인

#### 3.5 Message 수정
- [ ] 메시지 목록에서 "수정" 버튼 클릭
- [ ] 모달에서 내용, 요일, 시간, 웹훅 수정
- [ ] "수정" 버튼 클릭
- [ ] 목록에 변경사항 반영 확인

#### 3.6 Message 삭제
- [ ] 메시지 목록에서 "삭제" 버튼 클릭
- [ ] 확인 다이얼로그에서 확인
- [ ] 목록에서 제거 확인
- [ ] Firestore에서 삭제 확인

### 시나리오 4: 스케줄러 동작 검증

#### 4.1 스케줄러 설정
- [ ] 현재 시간 확인 (예: 14:30)
- [ ] 메시지 생성:
  - 요일: 오늘 요일 선택
  - 시간: 현재 시간 + 1분 (예: 14:31)
  - 활성화: 활성
- [ ] 메시지 저장 확인

#### 4.2 자동 전송 대기
- [ ] 1분 대기
- [ ] Mattermost 채널에서 메시지 수신 확인
- [ ] 메시지 내용이 정확한지 확인
- [ ] Firebase Functions 로그 확인

#### 4.3 여러 메시지 동시 전송
- [ ] 같은 시간에 여러 메시지 생성
- [ ] 모두 활성화
- [ ] 1분 대기
- [ ] 모든 메시지가 전송되는지 확인

#### 4.4 비활성 메시지 미전송 확인
- [ ] 비활성 메시지 생성
- [ ] 시간이 되었을 때 전송되지 않는지 확인
